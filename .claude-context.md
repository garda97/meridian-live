## 🔄 How It Works

**When bot executes LP transaction:**

```
sendAndConfirmTransaction(connection, tx, signers, options)
  └─ Wrapper checks config.jito.enabled
     ├─ TRUE:
     │   └─ await sendJitoBundle(connection, tx, signers, log)
     │       ├─ Fetch random tip account from Jito API
     │       ├─ Create tip transaction (0.001 SOL)
     │       ├─ Bundle: [tipTx, actualTx]
     │       ├─ POST to Jito Block Engine
     │       ├─ SUCCESS: return bundleId, log "[jito] Bundle accepted"
     │       └─ ERROR: throw → caught by wrapper
     └─ FALLBACK (on error OR if disabled):
         └─ return await sendAndConfirmTransaction_original(connection, tx, signers, options)
            └─ Standard RPC submission (original behavior)
```

## ⚙️ Configuration

### Environment Variables (Optional Overrides)
```bash
# Override Jito endpoint (default: mainnet NY)
export JITO_BLOCK_ENGINE="https://ny.mainnet.block-engine.jito.wtf"

# Override tip amount in lamports (default: 1_000_000 = 0.001 SOL)
export JITO_TIP_LAMPORTS="1000000"
```

### Toggle Jito On/Off (Without Restart)
Edit `/opt/meridian/config.js`:
```bash
# DISABLE Jito (revert to standard RPC only)
sed -i 's/enabled: true,/enabled: false,/' /opt/meridian/config.js

# ENABLE Jito
sed -i 's/enabled: false,/enabled: true,/' /opt/meridian/config.js
```

After toggling, the next cron cycle (~10 min) will reload config. Or restart manually:
```bash
pkill -f 'node /opt/meridian/index.js'
sleep 2
cd /opt/meridian && nohup node index.js > /tmp/meridian.log 2>&1 &
```

## 🔐 Security Notes

- **Jito tip accounts:** Fetched from Jito API at runtime (not hardcoded) — prevents address mistyping
- **Private keys:** Stored in encrypted env (envcrypt.js), not readable via normal file ops
- **Bundle serialization:** Uses bs58 standard (not custom byte encoding) — battle-tested format
- **Error handling:** 5s timeout prevents hung transactions
- **Fallback:** RPC submission always available, no single point of failure
