ALERT TYPES IMPLEMENTED
================================================================================

1. Position Closure          ❌  Triggered when position closes
2. PnL Change              📈  Gain >0.5% increase
3. PnL Loss                📉  Loss >0.5% decrease  
4. Out-of-Range Status     ⚠️  Position moves out of range
5. Back In-Range           ✅  Position returns to range
6. Fee Accumulation        💰  >$5 accumulated in unclaimed fees
7. Trailing Stop           🎯  Trailing stop activated
8. Stop Loss Hit           🚨  Position hits -15% or worse

Alert format: [POOL-NAME] Emoji Alert_Type | Details

Examples:
  ❌ [RTM-SOL] Position CLOSED | pumped far above range
  📈 [world-SOL] PnL +3.2% | Δ +2.8%
  ⚠️ [PEACE-SOL] OUT OF RANGE | Perlu rebalance
  💰 [NYAN-SOL] Fees +$12.50 | Ready to claim
  🎯 [RTM-SOL] Trailing Stop ACTIVATED
  🚨 [FROGBULL-SOL] [STOP LOSS] hit -16.5%

================================================================================
MONITORING WORKFLOW
================================================================================

Every 5 minutes (once cron is installed):

  1. Cron trigger
       ↓
  2. Load current state.json
       ↓
  3. Compare against /tmp/meridian_monitor_state.json
       ↓
  4. Detect material changes
       ├─ Position closures
       ├─ PnL changes >0.5%
       ├─ Out-of-range status changes
       ├─ Fee accumulation >$5
       ├─ Trailing stop activation
       └─ Stop loss hits <-15%
       ↓
  5. Save new snapshot for next cycle
       ↓
  6. Append summary to MONITOR.md
       ↓
  7. If alerts detected + TELEGRAM_BOT_TOKEN set:
       ├─ Parse alerts
       ├─ Send to Telegram API
       └─ Log delivery status
       ↓
  8. Exit (0=alerts sent, 1=silent)

TROUBLESHOOTING
================================================================================

PROBLEM: Monitor not sending alerts to Telegram
  → Check: grep TELEGRAM /root/meridian/.env
  → Test: curl -X POST "https://api.telegram.org/botTOKEN/sendMessage" ...
  → Fix: Add TELEGRAM_BOT_TOKEN to .env

PROBLEM: Cron job not running
  → Check: sudo service cron status
  → View: crontab -l | grep meridian
  → Verify: tail -20 /var/log/meridian_monitor.log

PROBLEM: Position data not updating
  → Check: cat /root/meridian/state.json | head -20
  → Verify: Meridian daemon is running and updating state

PROBLEM: Too many false alerts
  → Edit thresholds in monitor_positions.py (lines with >0.5, >5, <-15)
  → Test changes with: python3 scripts/monitor_positions.py
